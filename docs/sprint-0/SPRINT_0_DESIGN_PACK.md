# Sprint 0 Design Pack

## Purpose
Sprint 0 is not a feature sprint.
It does not build the product.
It prepares the repository for disciplined coding under the fixed MVP implementation posture.

Its purpose is to establish:

- repository structure
- naming rules
- shared contract schema pack
- testing skeleton
- coding rules
- first implementation boundary
- admission criteria for Sprint 1

## Scope
Sprint 0 may define only:

- structure
- file and folder boundaries
- shared schemas
- interface contracts
- test harness skeleton
- repository conventions
- implementation discipline

Sprint 0 must not define:

- advanced UI/UX behavior
- product polish
- hidden policy logic
- advanced educational behavior
- rich calibration logic

## Repository posture
Official posture:

**Constrained Core Build**

Meaning:

- architecture-first repository
- no mixed-responsibility files
- explicit module boundaries
- deterministic flow where possible
- typed shared contracts
- traceable outputs
- no hidden logic in interface layer

## Sprint 0 deliverables
By the end of Sprint 0, the repository must have:

- repository structure
- naming doctrine
- shared contract schema pack
- testing skeleton
- coding rules
- initial file map
- first milestone definition
- done criteria for Sprint 1 admission

## Initial milestone boundary
Sprint 0 does not fully implement the first MVP milestone.
Sprint 0 prepares the conditions for the first implementation nucleus:

single-message end-to-end analytical pass

## Task groups
### A. Repository structure
- create top-level tree
- create src, tests, docs, scripts, examples

### B. Shared contracts
- input contracts
- analysis contracts
- taxonomy contracts
- risk contracts
- output contracts

### C. Testing skeleton
- pytest config
- empty unit tests
- contract test shells
- integration test shell
- fixtures folder

### D. Governance docs
- repository map
- naming rules
- coding rules
- testing skeleton doc

### E. App nucleus shell
- single_message_pass/run.py
- pipeline.py
- empty orchestration path

## Definition of done
Sprint 0 is done only if all of the following are true:

1. The repository has an architecture-first tree.
2. A shared contract schema pack exists.
3. A testing skeleton exists.
4. Explicit coding rules exist.
5. A documented repository map exists.
6. A single-message pipeline shell exists.
7. No hidden product logic is introduced.
8. Core, output, and education responsibilities remain separated.

## Sprint 1 admission criteria
Sprint 1 may start only if Sprint 0 closes with:

- repository tree in place
- shared contracts present
- test harness present
- module placeholders created
- initial docs present
- app nucleus shell present
