# Archive

Status: Post-MVP archive root.

## Purpose

This folder preserves historical Cognitive Shield project material that should no longer remain in the active documentation path but should not be deleted.

Archive does not mean invalid.

Archive does not mean obsolete.

Archive means preserved for traceability, historical context, and future review.

## What Belongs Here

This archive area may later contain:

- pre-MVP historical material;
- old sprint material;
- replaced documents;
- legacy planning documents;
- historical architecture notes;
- superseded working drafts.

## What Does Not Belong Here

The following should not be moved here without separate review:

- active MVP proof documents;
- current repo-governance documents;
- current source files;
- current test files;
- current workflow files;
- current README or license files.

## Movement Rule

No files should be moved into this archive without a separate archive movement boundary review.

## Protected MVP Baseline

The Sprint 4 MVP functional proof baseline remains protected.

Archive movement must not damage:

- MVP functional proof;
- smoke-test validation;
- Python Tests workflow;
- post-merge verification;
- MVP status documentation.
