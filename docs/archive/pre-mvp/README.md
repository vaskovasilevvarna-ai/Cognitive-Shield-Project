# Pre-MVP Archive

Status: Post-MVP archive subfolder.

## Purpose

This folder is reserved for historical material from before the Sprint 4 MVP functional proof baseline.

The purpose is to preserve pre-MVP context without keeping every historical document in the active documentation path.

## Possible Future Contents

This folder may later contain:

- early architecture records;
- early sprint material;
- pre-MVP planning artifacts;
- historical design notes;
- early repository setup notes;
- documents that explain how the project reached MVP proof.

## What Does Not Belong Here

This folder should not contain:

- active Sprint 4 MVP proof documents;
- current source files;
- current test files;
- current workflow files;
- current repo-governance documents;
- production or release documents.

## Movement Rule

No files should be moved here until archive movement is separately reviewed and admitted.

## Interpretation

Pre-MVP archive does not mean the content is unimportant.

It means the content belongs to the historical path before MVP-level functional proof was achieved.
